# Ant Design Vue学习笔记

## Result 结果

### 1.何时使用

当有重要操作需告知用户处理结果，且反馈内容较为复杂时使用。

#### 常见使用场景

| 场景         | 推荐 status | 典型操作按钮             |
| :----------- | :---------- | :----------------------- |
| 表单提交成功 | `success`   | "返回列表"、"继续添加"   |
| 支付成功     | `success`   | "查看订单"、"继续购物"   |
| 注册完成     | `success`   | "去邮箱验证"、"返回登录" |
| 操作失败     | `error`     | "重试"、"联系客服"       |
| 页面不存在   | `404`       | "返回首页"               |
| 无权限访问   | `403`       | "申请权限"、"返回上一页" |

### 2.什么是 a-result？

`a-result` 是 **Ant Design Vue** 组件库中的一个反馈组件，专门用于**操作结果页面**（如提交成功、支付失败、注册完成等场景）。它提供了一个标准化的视觉模板，帮助用户快速理解操作的结果以及接下来的步骤。

简单来说：当你需要告诉用户“操作成功了”或“操作失败了”时，就可以用 `a-result`。

### 3. 可用的 status 类型

| status 值 | 图标样式       | 适用场景                           |
| :-------- | :------------- | :--------------------------------- |
| `success` | ✅ 绿色对勾     | 操作成功（提交、支付、注册完成等） |
| `error`   | ❌ 红色叉号     | 操作失败（网络错误、验证不通过等） |
| `info`    | ℹ️ 蓝色信息图标 | 提示类信息                         |
| `warning` | ⚠️ 黄色警告图标 | 警告类信息                         |
| `404`     | 404 图标       | 页面未找到                         |
| `403`     | 403 图标       | 权限不足                           |
| `500`     | 500 图标       | 服务器错误                         |

### 4.组件属性（Props）

| 属性名      | 说明                | 类型          | 默认值 |
| :---------- | :------------------ | :------------ | :----- |
| `title`     | 结果标题            | string / slot | -      |
| `sub-title` | 结果副标题/补充说明 | string / slot | -      |
| `status`    | 结果状态            | string        | 'info' |
